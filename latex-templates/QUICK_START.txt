================================================================================
  STEP 3: CONFIGURE AUTHENTICATION
================================================================================

Option A - SSH Keys (Recommended):

  ssh-keygen -t ed25519 -C "your.email@example.com"
  cat ~/.ssh/id_ed25519.pub

  Copy output and add to GitHub:
    https://github.com/settings/keys → "New SSH key"

  git remote set-url origin git@github.com:YOUR_USERNAME/latex-templates.git

Option B - Personal Access Token:

  Generate at: https://github.com/settings/tokens
  Select scope: "repo" (all)
  Use token as password when pushing

================================================================================
  STEP 4: PUBLISH TEMPLATES TO GITHUB
================================================================================

Choose ONE of these methods:

Method A - Publish Everything (First Time):

  git add templates/ output/
  git add COMPILATION_REPORT.md TEMPLATE_SUMMARY.md
  git status  # Review what will be committed
  ./scripts/publish_all.sh

Method B - Publish By Category:

  ./scripts/publish_category.sh aerospace
  ./scripts/publish_category.sh physics
  # ... continue with other categories

Method C - Manual Control:

  git add templates/
  git add output/
  git commit -m "Add LaTeX template collection"
  git push -u origin main

================================================================================
  STEP 5: COCALC SHARESERVER INTEGRATION (OPTIONAL)
================================================================================

Find your CoCalc Project ID from the browser URL:
  https://cocalc.com/projects/YOUR-PROJECT-ID/files/...

Generate share links:

  ./scripts/generate_share_links.sh YOUR-PROJECT-ID
  cat SHARE_LINKS.md  # View generated links

Publish to GitHub:

  git add SHARE_LINKS.md
  git commit -m "Add CoCalc ShareServer links"
  git push

Enable public access in CoCalc:

  1. Open CoCalc project
  2. Settings → Public files
  3. Enable: templates/ and output/
