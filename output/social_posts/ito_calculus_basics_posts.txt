TITLE: I built a Python simulation to understand why stochastic integrals are "broken" (Itô calculus)

BODY:

**The Problem**

In regular calculus, we learn that ∫x dx = x²/2. Simple, right?

But when x is replaced by Brownian motion W_t (the mathematical model of random noise), something weird happens:

∫W_t dW_t = ½W_T² - **½T**

Where does that extra -½T come from?!

**The Answer: Quadratic Variation**

Brownian motion is so "rough" that it accumulates variation at rate T over time interval [0,T]. This affects how we compute integrals. The Itô integral uses left-point evaluation (like W_t at time t_i), while the Stratonovich integral uses midpoints.

The difference between them is exactly T/2 - the quadratic variation.

**Why This Matters**

This is the foundation of mathematical finance. The Black-Scholes formula for option pricing uses Itô calculus, and that mysterious -σ²/2 drift correction comes directly from Itô's lemma.

**What I Learned**

I simulated 5000 Brownian motion paths and computed both integrals numerically:

- Itô integral mean: ≈ -0.5 (matches theory!)
- Stratonovich integral mean: ≈ 0 (ordinary calculus result)
- Itô isometry: E[(∫f dW)²] = ∫f² dt verified for multiple test functions

The code uses NumPy for vectorized simulation and Matplotlib for visualization. No external dependencies beyond the scientific Python stack.
