Just built a QR decomposition using Givens rotations from scratch!

The idea is beautifully simple: use 2D rotations to zero out matrix elements one by one. Each rotation only touches two rows, making it perfect for sparse matrices.

The clever part? Computing c and s to zero element b:
- If |b| > |a|: τ = -a/b, s = 1/√(1+τ²), c = s·τ
- Otherwise: τ = -b/a, c = 1/√(1+τ²), s = c·τ

This avoids overflow/underflow. Math is beautiful!

Implemented Givens rotations for QR decomposition. These are n×n orthogonal matrices that rotate in a 2D subspace (i,j plane).

Key properties:
• Gᵀ G = I (orthogonal)
• det(G) = 1 (proper rotation)
• Only affects components i and j

Complexity: O(mn²) for m×n matrix - more expensive than Householder but better for:
- Sparse matrices (preserves structure)
- Parallel architectures
- Incremental updates

The numerically stable formulation using τ = min/max avoids computing √(a² + b²) directly.

**Title:** Implemented QR Decomposition Using Givens Rotations - Here's How It Works

**Body:**

I've been exploring numerical linear algebra methods and just implemented QR decomposition using Givens rotations. Here's an ELI5 explanation:

**What's a Givens Rotation?**

Imagine you have a vector [3, 4]. A Givens rotation finds the angle θ that rotates this vector onto the x-axis, giving [5, 0]. The rotation matrix is:

```
[cos(θ)   sin(θ) ]
[-sin(θ)  cos(θ)]
```

**Why use them for QR?**

To decompose A = QR, we need R to be upper triangular (zeros below diagonal). Givens rotations let us zero out one element at a time by rotating pairs of rows.

**The numerically stable trick:**

Instead of computing √(a² + b²) directly (which can overflow), we use:
- If |b| > |a|: τ = -a/b, then s = 1/√(1+τ²), c = s·τ

This keeps τ ≤ 1, avoiding numerical issues.

**Key findings:**
- Applied 6 rotations to decompose a 4×3 matrix
- ||A - QR|| ≈ 10⁻¹⁵ (machine precision!)
- ||QᵀQ - I|| ≈ 10⁻¹⁵ (Q stays orthogonal)

**When to use Givens over Householder:**
- Sparse matrices (Givens preserves sparsity)
- Parallel computing (rotations on different rows are independent)
- Updating existing factorizations
